features

Eleven diagnostic surfaces, one admin page.

01 environment

PHP, server, WordPress runtime

PHP version and SAPI, memory and execution limits, extension presence, server software, WP config, multisite, cron mode.

02 filesystem

Permissions, ownership, disk

Per-directory permission, writability and ownership checks for ABSPATH, wp-content, AI1WM storage and backup paths, plus disk space.

03 database

Schema, sizes, autoload

MySQL or MariaDB version, charset, collation, table prefix, per-table engine and size, prefixed vs orphaned tables, autoloaded options weight.

04 plugins

Conflicts and AI1WM ecosystem

All active and inactive plugins with versions, full AI1WM ecosystem scan with installed vs latest comparison, known-conflict detection.

05 real-time log

Live export and import stream

Six logging channels (pipeline, params, status, exclusions, HTTP, errors). Per-run log files. Filter overrides for chunk size, retries, timeouts.

06 support access

Time-boxed tokens with audit log

Debug Only or Full Admin tokens, SHA-256 hashed, rate-limited, 72h expiry, email notification on use, complete session audit trail.

install

Three steps. No external dependencies.

$ wp-admin / install-plugin
  1. step 01

    Download

    Click the Download button above to grab servmask-debug.zip from the latest release.

  2. step 02

    Upload

    In WordPress: Plugins > Add New > Upload Plugin. Choose the ZIP and install.

  3. step 03

    Activate

    Activate the plugin, then open ServMask Debug in the admin sidebar to begin diagnostics.
